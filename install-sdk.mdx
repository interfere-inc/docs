The `@interfere/next` package is the official Interfere SDK for Next.js applications. It instruments your app with error tracking, session replay, analytics, and page-event capture, and streams that data to Interfere so issues can be detected, triaged, and explained in real time. This page covers everything from installation to verifying that your connection is live.

## Installation

Add `@interfere/next` to your project using npm, yarn, or pnpm.

<CodeGroup>

```bash npm
npm install @interfere/next
```

```bash yarn
yarn add @interfere/next
```

```bash pnpm
pnpm add @interfere/next
```

</CodeGroup>

<Note>
  `@interfere/next` supports Next.js 13 and later with the App Router. If you are using the Pages Router, check back for updates or contact the Interfere team.
</Note>

## Adding InterfereProvider to your layout

`InterfereProvider` is a React context provider that initializes the SDK and makes its features available throughout your application. You should mount it once, at the root of your component tree, inside `app/layout.tsx`.

### Basic usage

Import `InterfereProvider` from `@interfere/next/provider` and wrap your `{children}` with it. Configure the provider with your project credentials as shown in **Settings → Project** in your Interfere workspace.

```tsx app/layout.tsx
import { InterfereProvider } from "@interfere/next/provider";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>
        <InterfereProvider>
          {children}
        </InterfereProvider>
      </body>
    </html>
  );
}
```

## Configuration

### Credentials and environment variables

Store your project credentials in environment variables rather than hardcoding them, so they stay out of source control. Retrieve the values from **Settings → Project** in your Interfere workspace and add them to your `.env.local` file.

<Warning>
  Never commit your `.env.local` file to source control. Add it to your `.gitignore` if it isn't already there.
</Warning>

Each workspace has unique credentials. If you want to keep issue inboxes separate across environments, create a distinct Interfere project for development, staging, and production and configure each environment with its own values.

### InterfereProvider props

| Prop | Required | Type | Description |
|------|----------|------|-------------|
| `consent` | No | `{ analytics?: boolean; replay?: boolean }` | Controls which SDK features activate based on user consent preferences. See [Consent Management](#consent-management) below. |
| `children` | ✅ Yes | `React.ReactNode` | Your application's component tree. |

## Consent management

Interfere supports granular consent gating so you can respect your users' privacy preferences and stay aligned with audit frameworks like SOC 2 Type II and GDPR.

Pass a `consent` prop to `InterfereProvider` to control which features run. Error tracking is always active — it is treated as a necessary function. Analytics (page events, rage clicks, fingerprint) and session replay are independently toggleable.

```tsx app/layout.tsx
<InterfereProvider
  consent={{ analytics: true, replay: false }}
>
  {children}
</InterfereProvider>
```

### Reactive consent updates with consent.set()

When a user updates their preferences through your consent banner, call `consent.set()` and the SDK will load or tear down plugins in real time — no page reload required.

```tsx
import { consent } from "@interfere/next";

// Call this when the user accepts or changes their consent preferences
consent.set({ analytics: true, replay: true });
```

### Integrating with a consent management platform

If you use a consent management platform (CMP) such as c15t, CookieYes, OneTrust, or your own implementation, you can read consent state dynamically and pass it to `InterfereProvider`. The SDK will load or tear down plugins in real time whenever the `consent` prop value changes.

```tsx app/layout.tsx
import { InterfereProvider } from "@interfere/next/provider";

export default function Layout({ children }: { children: React.ReactNode }) {
  const { has } = useConsentManager(); // from your CMP

  return (
    <InterfereProvider
      consent={{ analytics: has("measurement"), replay: has("experience") }}
    >
      {children}
    </InterfereProvider>
  );
}
```

<Tip>
  Consent state is reactive — there is no need to reload the page when a user updates their preferences. The SDK responds to prop changes immediately.
</Tip>

## Verifying the connection

Once `InterfereProvider` is in place, start your development server:

```bash
npm run dev
```

Open your application in the browser and navigate to a few pages to generate activity. Then open the Interfere desktop app or the web dashboard. You should see your project marked as **Connected** and incoming data reflected in the session feed within a few seconds.

If you do not see a connection:

1. Open your browser's developer console and look for any errors prefixed with `[Interfere]`.
2. Confirm your project credentials match the values shown in **Settings → Project** exactly — watch out for leading or trailing whitespace.
3. Ensure `InterfereProvider` wraps the part of the tree where you expect activity to be captured. It must be a client-side component boundary; if your root layout is a Server Component, the provider handles this internally, but double-check that no intermediate Server Component is stripping the context.

<Note>
  Issues detected in development are scoped to your project the same way production issues are. If you want to keep dev and prod inboxes clean, create a separate Interfere project for local development and use its credentials in `.env.local`.
</Note>
