Interfere's developer platform lets you create OAuth applications and generate API keys with granular scopes, so you can extend Interfere into the rest of your stack. Whether you're building a lightweight internal integration, wiring up a custom dashboard, or automating workflows based on issue data, the API gives you a stable foundation to build on. This is the beginning of Interfere's broader platform strategy — more capabilities are coming soon.

<Note>
  OAuth applications and scoped API keys are currently in **private beta**. Access is invite-only while we work closely with early partners to refine the developer experience, endpoints, and tooling. See [Getting access](#getting-access) below to request early access.
</Note>

## What are OAuth applications and API keys?

An **OAuth application** represents your integration — it holds a name, description, and the permission scopes it requires. Once you create an application, you can generate one or more **API keys** tied to it. Each key inherits the scopes you defined, ensuring it can only access exactly what it needs.

**Scopes** are granular permissions that control what an API key can read or write within Interfere. Instead of issuing an all-or-nothing key, you choose the specific scopes that match your use case. For example, a read-only reporting dashboard only needs read scopes, while an automated workflow that updates issue status needs the appropriate write scope.

## Creating an OAuth application

<Steps>
  <Step title="Open the Developer settings">
    In the Interfere dashboard, navigate to **Settings → Developer**. This section is visible to workspace owners and admins.
  </Step>
  <Step title="Create a new application">
    Click **New OAuth Application**. Give your application a name that describes what it does (for example, `Internal Status Dashboard` or `PagerDuty Sync`), and optionally add a description and a redirect URI if your integration uses the OAuth authorization flow.
  </Step>
  <Step title="Select your scopes">
    Choose the scopes your application requires. Scopes are grouped by resource type, with separate read and write permissions for each. Grant only the scopes your integration actually needs.

    <Tip>
      Following the principle of least privilege keeps your integration safer. If a key is ever compromised, a narrowly scoped key limits what an attacker can access.
    </Tip>
  </Step>
  <Step title="Save the application">
    Click **Create Application**. Your application will appear in the developer settings list, ready for key generation.
  </Step>
</Steps>

## Generating API keys

Once your OAuth application exists, you can generate one or more API keys tied to it.

<Steps>
  <Step title="Open your application">
    From **Settings → Developer**, click on the application you just created.
  </Step>
  <Step title="Generate a new key">
    Click **Generate API Key**. Give the key a label so you can identify it later (for example, `Production` or `CI Pipeline`).
  </Step>
  <Step title="Copy and store the key securely">
    Your API key is shown **once** immediately after generation. Copy it and store it in a secrets manager or environment variable — you won't be able to view it again.

    <Warning>
      Never commit API keys to source control or expose them in client-side code. Treat them like passwords.
    </Warning>
  </Step>
  <Step title="Use the key in your integration">
    Pass your API key in the `Authorization` header of every request:

    ```http
    Authorization: Bearer YOUR_API_KEY
    ```

    Example request using cURL (replace the URL with the endpoint provided in your API documentation):

    ```bash
    curl https://api.interfere.com/v1/YOUR_ENDPOINT \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Accept: application/json"
    ```
  </Step>
</Steps>

## Understanding scopes

Scopes follow a `resource:action` format. Each scope controls access to a specific resource and operation within Interfere — for example, reading issues versus writing to them. When you create an application, the dashboard presents the full list of available scopes grouped by resource type.

<Note>
  The complete scope list is provided in the developer settings UI during setup and is actively evolving during the private beta. If you need a scope that isn't currently available, reach out to the Interfere team — we'd love to understand your use case.
</Note>
