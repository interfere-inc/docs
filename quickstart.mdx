---
title: "Quickstart: detect your first issue with Interfere"
sidebarTitle: "Quickstart"
description: "Get from zero to your first detected issue in under five minutes by installing @interfere/next and wrapping your Next.js app with InterfereProvider."
---

This guide walks you through everything you need to get Interfere running inside a Next.js application. By the end you'll have the SDK installed, your app wrapped with `InterfereProvider`, and a live issue appearing in your Interfere inbox.

<Steps>

  <Step title="Join the waitlist and get access">
    Interfere is currently in early access. Head to [interfere.com/auth/waitlist](https://interfere.com/auth/waitlist), enter your email address, and you'll receive an invitation when your spot is ready.

    Once you receive your invitation, follow the link in the email to sign in and create your workspace. Your workspace settings will include the project credentials you need to configure the SDK.

    <Tip>
      Already been invited? Go straight to [interfere.com/auth](https://interfere.com/auth) to sign in and grab your project credentials from **Settings → Project**.
    </Tip>
  </Step>

  <Step title="Download the Interfere desktop app">
    The Interfere desktop app is where your issue inbox lives. Download it for your operating system from [interfere.com/download](https://interfere.com/download).

    | Platform | Download |
    |----------|----------|
    | macOS    | [interfere.com/download](https://interfere.com/download) → Download for macOS |
    | Windows  | [interfere.com/download](https://interfere.com/download) → Download for Windows |

    Open the app and sign in with the same account you used on the web. Your inbox will be empty for now — that changes once the SDK is connected.
  </Step>

  <Step title="Install the @interfere/next SDK">
    Add the `@interfere/next` package to your Next.js project using your preferred package manager.

    <CodeGroup>

    ```bash npm
    npm install @interfere/next
    ```

    ```bash yarn
    yarn add @interfere/next
    ```

    ```bash pnpm
    pnpm add @interfere/next
    ```

    </CodeGroup>

    <Note>
      `@interfere/next` requires Next.js 13 or later with the App Router. Pages Router support is not available at this time.
    </Note>
  </Step>

  <Step title="Wrap your app with InterfereProvider">
    Open your root layout file (`app/layout.tsx`) and import `InterfereProvider` from `@interfere/next/provider`. Wrap your `{children}` with the provider, passing your project credentials as described in **Settings → Project** in your workspace.

    ```tsx app/layout.tsx
    import { InterfereProvider } from "@interfere/next/provider";

    export default function RootLayout({
      children,
    }: {
      children: React.ReactNode;
    }) {
      return (
        <html lang="en">
          <body>
            <InterfereProvider>
              {children}
            </InterfereProvider>
          </body>
        </html>
      );
    }
    ```

    Store any credentials in environment variables so they stay out of source control. Refer to the [Install SDK](/install-sdk) guide for full configuration options, including how to control which SDK features activate based on user consent.
  </Step>

  <Step title="Verify your first issue appears in the inbox">
    Start your development server and navigate around your app to generate some activity.

    ```bash
    npm run dev
    ```

    Switch to the Interfere desktop app. Within a few moments you should see connection activity appear. Once Interfere begins receiving data from your app, any detected anomalies will surface as issues in your inbox.

    <Tip>
      Don't see anything yet? Check the browser console for any errors prefixed with `[Interfere]` and confirm your project credentials match exactly what appears in **Settings → Project** in the Interfere web app.
    </Tip>

    You're all set. From here, explore how Interfere triages and explains issues in [Concepts: Issues](/concepts/issues), or read the full SDK reference in the [Next.js SDK](/sdk/next-js) guide.
  </Step>

</Steps>
