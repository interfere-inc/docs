Interfere is built with security as a first principle, not an afterthought. From the way the SDK collects data to how your team accesses the dashboard, every layer of the product is designed to minimize exposure, respect user consent, and give you full visibility into what Interfere knows about your users. This page outlines our current compliance posture, the data we collect, and the controls you have over that collection.

### SOC 2 Type II

Interfere is officially in its SOC 2 Type II observation period. This audit examines our security, availability, and confidentiality controls over an extended window — not just a point-in-time snapshot — making it one of the most rigorous certifications available to SaaS companies. We expect to complete the audit and receive certification in Q2 2026.

### GDPR

GDPR compliance is actively in progress. Our work is structured around four core areas:

<AccordionGroup>
  <Accordion title="Data minimization and purpose limitation">
    Interfere only collects what is necessary to detect and triage issues. Our consent management system lets you gate each data collection category — analytics, session replay, and fingerprinting — so data is never gathered without a valid legal basis. Error tracking is treated as a necessary function and always runs; all other categories are independently toggleable.
  </Accordion>
  <Accordion title="User rights and transparency">
    We provide mechanisms for data subjects to exercise their rights, including access, rectification, erasure, and portability. Requests can be submitted to [legal@interfere.com](mailto:legal@interfere.com) and are processed in accordance with applicable data protection law.
  </Accordion>
  <Accordion title="Technical and organizational safeguards">
    Data is encrypted in transit and at rest. Access to production systems is restricted by role, protected with multi-factor authentication, and audited. We maintain incident response procedures and conduct regular internal reviews.
  </Accordion>
  <Accordion title="Consent management integration">
    The Interfere SDK integrates natively with major consent platforms — c15t, CookieYes, OneTrust — and supports any custom consent management provider. When a user updates their preferences, the SDK loads or tears down data collection plugins in real time.
  </Accordion>
</AccordionGroup>

## What Data Interfere Collects

Interfere collects data at the SDK level in order to detect, reproduce, and triage software issues. The exact data collected depends on which SDK features are active for a given user session.

| Category | Examples | Consent required |
|---|---|---|
| **Error events** | Exceptions, stack traces, error rates | No — treated as a necessary function |
| **Session events** | Page views, navigation, user interactions | Yes — requires analytics consent |
| **Session replay** | Recorded session footage, rage clicks | Yes — requires replay consent |
| **User identifiers** | User ID, anonymous fingerprint | Yes — requires analytics consent |

<Warning>
  If you operate under GDPR or another privacy regulation, you should configure consent management in your SDK integration before going to production. Without consent gating, Interfere may collect data beyond what your users have agreed to.
</Warning>

### Consent Controls

The `InterfereProvider` accepts a `consent` prop that maps directly to data collection categories. Pass consent state from your existing consent management platform and Interfere will activate only the appropriate features.

```tsx
import { InterfereProvider } from "@interfere/next/provider";

export default function Layout({ children }: LayoutProps<"/">) {
  const { has } = useConsentManager(); // from your CMP

  return (
    <InterfereProvider
      consent={{
        analytics: has("measurement"),
        replay: has("experience"),
      }}
    >
      {children}
    </InterfereProvider>
  );
}
```

Consent state is reactive. When a user updates their preferences — through a consent banner, account settings, or a login event — call `consent.set()` and the SDK will immediately load or tear down the affected plugins without a page reload.
