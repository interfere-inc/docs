Every issue in Interfere is anchored to the real people experiencing it. Rather than treating problems as abstract error counts, Interfere connects each issue to the specific users who are affected — giving your team a precise answer to the question "who is this hurting, and how many of them are there?" User tracking is powered by the `identify()` call in the Interfere SDK, which links your application's user identity to the behavioral signals Interfere collects.

## The Impacted Users panel

The Impacted Users panel appears in the right sidebar of every Issue Detail view. It shows:

- **Total affected users** — the number of unique users who have encountered the issue since it was first detected (e.g., *12,881*)
- **Impacted users chart** — a timeline graph showing how the affected user count has grown or changed over time
- **User list** — a scrollable list of individual affected users with quick-access profile links

### Impacted users chart

The chart visualizes the trajectory of user impact from the moment the issue was first detected. A steep upward slope signals a rapidly spreading problem that should be escalated; a plateau or downward slope after a fix deployment confirms that the issue is resolving.

The x-axis represents time from first detection to the current moment. The y-axis shows the cumulative count of unique affected users. Deployment markers are overlaid on the chart when Interfere has connected the issue to a release, so you can see whether a deployment was the inflection point.

<Tip>
  Use the impacted users chart when deciding whether to escalate an issue's priority. An issue affecting 340 users that has plateaued is a different situation from one affecting 340 users that doubled in the last ten minutes.
</Tip>

## User profiles

Clicking any user in the Impacted Users list opens their full profile. A user profile brings together everything Interfere knows about that individual across all of their sessions and any issues that have affected them.

### Example: Alva Hartmann

| Field | Value |
|---|---|
| **Name** | Alva Hartmann |
| **Email address** | alva.hartmann@email.com |
| **Country** | United States |
| **Source** | Clerk |
| **Surfaces** | Website Frontend, iOS App |
| **First seen** | January 12, 2026 · 11:55 AM |
| **Last seen** | March 3, 2026 · 07:18 AM |
| **Session count** | 78 |
| **Problem count** | 3 |
| **Active problems** | 1 active out of 3 total |

The profile also lists every issue that has affected this user, with the current status of each. Alva's profile shows that she is impacted by `#120 Broken password reset link`, which is currently active.

### Sessions on a user profile

Below the profile metadata, the **Sessions** tab lists every session Interfere has recorded for this user, ordered by most recent first. Each session entry shows its ID, event count, error count, duration, and timestamp. Clicking a session opens the full replay, letting you trace exactly what Alva experienced during that visit.

### Problems on a user profile

The **Problems** tab lists all issues that have affected this user — both active and historical. This makes it easy to understand whether a user is experiencing a single isolated bug or is consistently running into multiple problems across your product.

## Looking up a specific user

You can find any identified user directly from the **Users** section of Interfere, accessible from the main navigation.

<Steps>
  <Step title="Navigate to Users">
    Click **Users** in the left sidebar to open the Users directory.
  </Step>
  <Step title="Search by name or email">
    Type the user's name or email address into the search bar. Results update as you type. Searching `alva.hartmann` will surface Alva Hartmann's profile immediately.
  </Step>
  <Step title="Open the profile">
    Click the user's name to open their full profile, including all sessions, active problems, and historical issues.
  </Step>
  <Step title="Navigate to affected issues">
    From the profile's **Problems** tab, click any issue title to jump directly to that Issue Detail view — with the user's sessions pre-filtered to show only sessions relevant to that issue.
  </Step>
</Steps>

You can also look up a user from within an issue. In the Impacted Users panel on the Issue Detail view, click any user's name or avatar to open their profile without leaving the issue context.

## How the identify() call powers user tracking

User data in Interfere comes from the `identify()` call in the Interfere SDK. When your application calls `identify()` with a user's details, Interfere associates all subsequent events, errors, and sessions from that browser or device with that specific user.

```typescript
interfere.identify({
  userId: "usr_01HX5K9Z3QBMWJ4NE8FRDCYTPV",
  email: "alva.hartmann@email.com",
  name: "Alva Hartmann",
});
```

Once identified, any issue Interfere detects that involves this user's sessions will automatically appear on their profile and in the Impacted Users panel for the relevant issue.

You can pass any additional properties that are useful for your team:

```typescript
interfere.identify({
  userId: "usr_01HX5K9Z3QBMWJ4NE8FRDCYTPV",
  email: "alva.hartmann@email.com",
  name: "Alva Hartmann",
  country: "United States",
  plan: "pro",
  organizationId: "org_acme",
});
```

Custom properties appear on the user profile and can be used to filter the impacted users list on an issue — for example, filtering to only `plan: enterprise` users to assess whether a bug is isolated to a specific tier.

<Note>
  The `identify()` call should be made as early as possible in your application's lifecycle — typically right after your authentication check resolves — so that all subsequent events are attributed to the correct user.
</Note>

## Privacy and consent

Interfere collects only the user data you explicitly pass to `identify()`. No additional personal data is inferred or scraped. The fields you provide determine what appears on user profiles.

Session replays and behavioral events are subject to your users' consent preferences. If a user has not consented to tracking or session recording, their events will not be captured and they will not appear in the Impacted Users panel.

For full details on configuring what data is collected and which features activate based on consent signals, see [Consent Management](/sdk/consent-management).

<Warning>
  If you operate under GDPR or similar data protection regulations, ensure that the user data you pass to `identify()` — including email addresses — is covered by your privacy policy and collected with appropriate legal basis. Interfere is a data processor acting on your instructions.
</Warning>
